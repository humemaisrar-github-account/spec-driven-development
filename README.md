# Spec Driven Development Monorepo

This repository contains a comprehensive collection of development phases showcasing spec-driven development methodologies. Each phase represents a different stage or aspect of a larger software development project.

## Repository Structure

### Phase 1: CLI Application Development
- Focus: Command-line interface application with contract testing
- Technologies: Python, pytest
- Features: Task management CLI with integration and unit tests

### Phase 2: Web Application Foundation
- Focus: Full-stack web application with authentication
- Technologies: JavaScript/Node.js, Next.js, FastAPI, PostgreSQL
- Features: Todo application with user authentication and CRUD operations

### Phase 3: Enhanced Web Application with AI Chatbot
- Focus: Advanced features and AI integration
- Technologies: JavaScript/Node.js, Next.js, FastAPI, PostgreSQL, OpenAI API
- Features: Chatbot integration, enhanced UI/UX, improved authentication
- Live Application: [https://hackathon-2phase3.vercel.app/](https://hackathon-2phase3.vercel.app/)

### Phase 4: Containerized Deployment
- Focus: Docker containerization and deployment strategies
- Technologies: Docker, Kubernetes, Helm
- Features: Containerized applications with orchestration configurations

### Phase 5: Advanced Features & Dapr Integration
- Focus: Advanced task management with Dapr (Distributed Application Runtime)
- Technologies: Python, Dapr, PostgreSQL, Docker, Kafka
- Features: Recurring tasks, event-driven architecture, advanced scheduling, microservices

## Getting Started

Each phase contains its own documentation and setup instructions in its respective directory. Refer to individual README files for specific details on running and contributing to each phase.

## Purpose

This monorepo demonstrates:
- Evolution of a software project through different phases
- Spec-driven development practices
- Different technology stacks and architectural decisions
- Testing strategies at various levels
- Deployment and scaling considerations
